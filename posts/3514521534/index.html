本文默认你已经配置好基本的 Electron 工具，没有的请自行百度。

鉴于 Electron 的核心模块中没有捆绑任何用于打包或分发文件的工具，因此需要使用额外的工具来打包应用程序。官方教程中打包您的应用程序处介绍了 Electron-Forge，但很快又认识到另一个打包工具。为防止混淆，这个打包工具新开一篇进行介绍与使用。

Electron 部分文件下载可能较慢，可在相关命令前直接添加以下内容后执行（以electron-builder --dir命令为例）（不保证所有命令前添加这个均不会出错）：

由于能力与需求的问题，文章重点放在 win 平台，其他平台的配置主要通过参阅他人经验完成。文中部分配置命名可能有所不同，请根据上下文判断。

由于 npm 镜像在国外，因此需要有足够的耐心以应对首次安装配置。文中部分命令处同时给出 npm 与 yarn 命令，使用 yarn 命令可能会获得一点点的提升（包括但不限于速度）。但其中仍存在一些步骤执行速度较慢。

构建其他平台的应用程序

【说明：此处仅作简单介绍，打包 win 之外平台应用程序不是本文重点】

为了让编译时少输入点指令，我们可以在项目的package.json中开头配置脚本。前面示例中"packwin": " electron-builder --win --x64",含义为：使用 electron-builder 构建 Windows 平台 64 位应用，其他常用平台配置参数如下：

后缀含义
–mac, -m, -o, –macos打包为 macOS 应用程序
–linux, -l打包为 Linux 应用程序
–win, -w, –windows打包为 Windows 应用程序
–x64打包为 x64 应用程序
–ia32打包为 ia32 应用程序（32 位）
–armv7l打包为 armv7l 应用程序（32 位）
–arm64打包为 arm64 应用程序
–dir创建预打包应用程序的文件夹，一般用于测试
–prepackaged, –pd预打包应用程序（可分发格式）的路径
–projectDir, –project项目目录的路径。，默认为当前项目目录
–config, -c配置文件路径。 默认为electron-builder.yml或js、js5
