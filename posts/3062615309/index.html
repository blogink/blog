真做事的智能体：本地 OpenClaw 部署流程教程

发表于2026-03-07|更新于2026-03-07|经验记录
|总字数:4.3k|阅读时长:15分钟|浏览量:

OpenClaw 的官网 上来就说明了它的特点：

The AI that actually does things.

它前身是 Clawdbot，后为规避版权风险而改名为 Moltbot，现用名为 OpenClaw，LOGO 是龙虾图案。它有一个重要特点便是拥有代理操作的能力（例如操作浏览器、查找文件、收发邮件等），而且还是开放源码的（MIT 协议）。

至少目前它还是非常火爆的，国内许多代部署的服务都有了（甚至还有人带着电脑去线下学习的）。这几日我的 AnythingLLM 软件抽风，借此机会来尝试一二。

这里是官方文档：https://docs.openclaw.ai/CN。官方文档有中文，但我个人建议先跟着英语版本来（更加及时）。

一些题外话：

这个标题有夸大的成分。从我个人的角度来说，这个东西想要得到好的体验，要么付费买 Token（而且消耗量不小），要么本地备一个好点的设备（开销也不小），否则真就是“玩一玩”的程度。它现在还没有正式版本，需要的权限比较多，如果管理不好很容易出现问题（已经有人遇到邮件清空、文件误删、被当“肉鸡”等问题了）。

如果你安装这程序真的必须需要有像腾讯那个“线下帮扶指导”的协助，那么你可能不适合用它，因为往后的配置、处理异常等会常伴发生（我觉得目前绝大多数人都用不到它，有需求的能自己解决或是选择更加完善的开发工具）。一定要亲自部署，跑在隔离环境里的确安全，但那样的话不如直接用 Chatbox 一类的工具，还更方便，响应速度还快。

环境配置

前置说明：

  • 为了提高灵活性，我们使用 NVM（Node Version Manager）管理 Node.js 版本，并使用 NPM 管理 node 包。
  • 鉴于现阶段大部分家用电脑都用 Windows 系统，因此这里我们以 Windows11 演示。模型服务由本地 Ollama 实现（目前 Ollama 自己也已经有 GUI 界面了，虽然能设置的不多），并可能使用到 Git（这两个程序的安装流程在之前的文章中已有提及，可自行搜索）。
  • 所有涉及路径设置的部分，若无特别说明，均需根据实际情况自行调整。

Hooks

告诉你 Hooks 是什么、怎样用、有什么用。

（官方文档）Hooks 提供了一个可扩展的事件驱动系统，用于响应智能体命令和事件自动执行操作。Hooks 从目录中自动发现，可以通过 CLI 命令管理，类似于 OpenClaw 中 Skills 的工作方式。

Enable hooks?（是否启用 Hooks？）

OpenClaw 附带三个自动发现的捆绑 hooks：

  • session-memory：当你发出 /new 时将会话上下文保存到智能体工作区（默认 ~/.openclaw/workspace/memory/）
  • command-logger：将所有命令事件记录到 ~/.openclaw/logs/commands.log
  • boot-md：当 Gateway 网关启动时运行 BOOT.md（需要启用内部 hooks）

我是全部都启用了的（使用空格选中前面的复选框）。

实用命令

注：部分命令可在交互式终端 TUI 中以斜杠开头执行，可能有一定延迟；斜杠开头的还可以在 WebUI 中执行。

命令 效果

配置命令：

命令 效果

openclaw tui 启动交互式终端界面

openclaw configure 重新进入配置向导，可用于修改模型与偏好

命令 效果

openclaw status --deep 进一步进行状态检测
openclaw logs 查看最近日志
