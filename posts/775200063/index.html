低显存也能 AI 绘图：SD WebUI Forge 配置与使用

发表于2024-11-04|更新于2025-07-05|经验记录
|总字数:3.4k|阅读时长:11分钟|浏览量:

SD-WebUI 拥有良好的扩展性、跨平台、图形化、开放源码等特点，成为了比较有名的图形化 AI 作图程序。模型方面，只要遵循相应的许可证，几乎没有使用限制（大多模型采用了 Creative ML OpenRAIL-M 或 Fair AI Public License，不过最终还需以模型作者的说明和许可证为准）。

虽然配置 SD Webui 可以劝退很多人，好在已经有爱好者铺路了，例如 绘世启动器，免去了很多繁琐的配置问题，进一步拉低了 Stable Diffusion 的部署门槛。

但环境配置归环境配置，硬件呢？以目前的情况来看，在 PyTorch 官方弄好免 CUDA 加速推理而改用 Triton 内核前，运行 SD 还是在 GPU 上效率高。虽然这程序已经可以在个人设备上部署，但并非人人都有至少 GTX1050Ti 的硬件。买个新的？也并非所有人都有这条件与机会。

真就只能冒着爆显存的风险等个数分钟出图吗？

SD Webui Forge

SD Webui 本地部署硬件要求高、占资源大，虽可以云端部署，但基本都不免费，而且本身还不能完整复刻工作流。Controlnet、Omost、Style2Paints 等项目的作者张吕敏博士也发现了这个问题，他因此开发了 stable-diffusion-webui-forge（为了方便，后面我们将其简称为“Forge”）。从项目介绍中可以得知，该项目旨在简化开发、优化资源管理、加快推理速度并研究实验功能。项目每 90 天与原来的 A1111 的 SD Webui 同步一次，或者在有重要修复的时候同步。

相比于广为人知的 SD Webui，Forge 有几个显著特点：

  • 性能优化：极大简化了逻辑，让 Forge 版本的性能有所提升，甚至让 4GB 显存的设备也能运行起 SDXL（按照其说法，配置越低，提升感知越明显。“找茬”可能要另当别论）。
  • 资源管理优化：在原版部署的时候，SD Webui 容易因插件冲突而启动失败，而 Forge 大幅提高各类插件的兼容性。
  • 新增的生成算法：Forge 版集成了一些新的采样器和其他生成算法，使得生成的图像质量更高。
  • 更好的兼容性：对于最新的配置或较旧的设备，Forge 版本都会尽可能提供更好的兼容性，确保用户能够充分利用其计算资源。
  • 预装部分插件：ControlNet、Z123、3D 图等插件，Forge 都将其进行了预装，开箱即用。

它的更新频率比较高，支持的东西也比较多（现在连 Flux 模型都支持了）。由于 Forge 的界面与 SD Webui 的界面基本一致，因此先前使用 SD Webui 的用户可以很方便地上手 Forge。

当然，如果你的设备配置比较好，没有特殊情况，那么建议继续用 SD-Webui，因为 Forge 对于配置较好的设备而言提升不够明显，而且部分插件在 Forge 上无法正常运行。

安装方法

Win 部署

对于 Win 用户，我建议直接用作者提供的一键整合包（本文此处链接最后更新时间：2024 年 12 月 20 日）。下载前请注意 CUDA 版本，如果你使用的是 NVIDIA 显卡，且打算使用 N 卡来运行该程序。下载时，你需要注意网络配置（例如使用 Watt Toolkit（Steam++）等程序）。

>>> 点此下载一键式软件包 （CUDA 12.1 + Pytorch 2.3.1） <<<

还有一些其他的版本（也是一键整合包）：

使用 CUDA 12.1 + Pytorch 2.3.1 <- Forge 推荐

使用 CUDA 12.4 + Pytorch 2.4 <- 最快，但 MSVC 可能损坏，xformers 可能无法工作

使用 CUDA 12.1 + Pytorch 2.1 < 旧版本，旧环境

N 卡用户的 CUDA 需要自行安装。安装了但不知道 N 卡 CUDA 版本的，可以在 CMD 中输入命令查看 cuda-version：

一般在这个终端打印的表格的第一行。CUDA 高版本可以兼容 Torch 低版本，但最好不要低太多，无论是从性能还是从兼容角度。

下载后，在磁盘中找一个合适的位置（目录不要有中文），将其解压。之后使用 update.bat 进行更新，再使用 run.bat 运行。每次使用前都建议运行整合包内的 update.bat，以修复早期版本的潜在错误。

不使用整合包也可以，使用与 SD-WebUI 相同的方法来安装 Forge 即可。此时 Git、Python 之类的环境需要你自行配置，Git Clone 该储存库，然后运行 webui-user.bat。

注：使用官方的一键包初始化时，除了需要下载必要组件外，可能还要从 Huggingface 上下载一个 realisticVisionV51_v51VAE.safetensor 模型。

这个模型体积较大，且因国内网络问题容易中断。因此使用官方整合包的，建议先下载这个模型放到【Forge 整合包】/webui/models/Stabel-diffusion目录下（【Forge 整合包】需要根据自己的实际情况加以变通）。

我的建议是用 Watt Toolkit 最新版本（截止本文完成前，它最新实际为测试版，但已经可以加速 Huggingface 网站），打开加速，然后将 https://huggingface.co/casque/realisticVisionV51_v51VAE/blob/main/realisticVisionV51_v51VAE.safetensors 复制粘贴到下载器中（例如 Motrix、IDM 之类的）。还嫌慢的，建议用抱脸国内镜像的 realisticVisionV51_v51VAE.safetensors 地址进行下载。

Linux

整合包目前只有 Win 版。Linux 下，使用与 SD-WebUI 相同的方法来安装 Forge 即可。此时 Git、Python 之类的环境需要你自行配置，Git Clone 该储存库，然后运行 webui-user.sh。

但你都选择接触 Linux 了，按照你的动手能力和搜索能力，这应该不成问题。

绘世启动器

（更新：2024 年 12 月 31 日秋葉 aaaki 已制作 Forge 整合包，想解压直用的 win 用户可以一步到位直接前往 【AI 绘画】SD-Forge 整合包发布 的视频简介处下载）

初始化启动器

一键包本身即可正常使用，此小节所述非必须流程，但对于 Windows 用户而言可以进一步降低上手难度。

首次运行需要下载一些必要的文件。但以国内的网络条件来说，连接一些国外网站体验如“坐牢”一般~~（当然，你把网络代理玩出花也不是不可）~~。前面我们提到了秋葉 aaaki 开发的“绘世启动器”，它同样也可以用于管理 Forge；同时，由于该启动器比较“傻瓜式”，上手比较方便，故在此一并介绍。

这里以 Forge 整合包为例（启动器初始化下载的文件不包含 SD 本体，最终还是要配置下载，不如提前先准备好整合包，省去很多未知问题。嫌速度慢建议找个镜像获取链接后丢到 Motrix 中）。如果你使用其他整合包，请灵活参考。

下载整合包后，将其解压到磁盘中。在这里，我们假设整合包在 E 盘根目录中，整合包文件夹名称为SD-Webui-Forge，下文亦然。如果下载文件正确，解压后的目录应大致如下：

将前面下载好的绘世启动器放到webui目录中，并双击运行，等待下载必要组件后自动打开启动器界面。

整合包环境设置

在左侧菜单栏中下拉，找到齿轮形状按钮设置并点击，将右侧配置模式设置为专家。然后拉到最下面，将 Python 路径覆盖和 Git 路径覆盖设置为 Forge 整合包环境的实际路径（在整合包的 system 文件夹中。整合包 Python 路径为E:\SD-Webui-Forge\system\python\python.exe\system\python\python.exe，整合包 Git 环境为E:\SD-Webui-Forge\system\git\bin\git.exe）。

（可选）将网络设置处的代理设置中代理服务器地址填上（也可以不填，这样会自动检测系统代理并使用，而不是指定端口）。启动器可以设置国内镜像下载，但如果后面的过程中发生网络问题（例如控制台出现 SSL 连接失败等自样），请先终止进程，设置代理地址及代理工具后再次启动（或者暂时关闭设备中所有代理软件、加速器等工具）。

初始化 Forge

在绘世启动器的高级选项中选择生成引擎。这里需要根据自己的实际情况进行选择，能用由 Forge 决定的就不必再改。如果设置了不支持的生成引擎（驱动版本过低、Torch 版本不匹配等也算），后续一键启动会发生报错。

进入版本管理，点击右侧切换分支按钮，选择相应的远程地址。一般我们选择原版lllyasviel/stable-diffusion-webui-forge选项，然后点击下拉菜单中的切换分支按钮，在弹窗中确认即可。（你看到的修改后的远端地址可能是一个 GitCode 地址，这是启动器自动设置的，无需担心是不是配置出错，把它理解为国内加速地址就行了）。

完成后点击一键更新，自动同步最新版本。之后每次启动都建议来这里检查一下，实际上是代替了 Forge 官方一键包里面的更新脚本。

先别动 extensions，因为有些在 SD-WebUI 中能用的插件在 Forge 中不一定兼容。完成后回到一键启动，等待程序初始化。期间可能会连接到未被镜像的国外地址，可能会因为网络问题而出错，这里请自行想办法解决，有“魔法”就挂上，有加速器就用，提示 SSL 挂了什么的就关，多试几次、多重启几下，总会成功的。一切完成后，如果配置无误，则启动器会自动打开浏览器访问地址（默认为http://127.0.0.1:7860），若组件加载正常且可以正常使用，则表明基本配置已经设置完成。

设置

此节在 Forge Webui 中操作。大多数设置与 SD-Webui 相同或相似，使用过 SD-Webui 的，有些配置方法可以直接迁移过来。

语言

待 Forge Webui 加载完成后，在选项卡找到Settings，点击后在左侧菜单栏找到User interface并点击。回到上方，将Localization设置为zh-Hans (Stable)。没有加载出来可以点击右侧的刷新图标后尝试。设置完后点击Apply settings按钮，然后再点击Reload UI重载 UI 应用更改。

插件

建议去 Forge 的 Webui 界面里下载，而不是直接将原来 SD-Webui 的 extensions 文件夹里的东西直接复制过来，否则部分插件可能会因为兼容性问题而无法正常加载。

在 webui 的 extensions 选项卡中选择 Install from URL（从网址安装），在 URL for extension’s git repository 中填写相应的仓库地址，等待安装。期间如果出现网络不佳等问题，请自行解决。
