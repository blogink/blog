在本地部署 Deepseek-R1 模型并使用

发表于2025-02-06|更新于2025-03-01|经验记录
|总字数:2.5k|阅读时长:7分钟|浏览量:

今年春节，大伙应该了解到，国产深度学习模型 DeepSeek 话题热度剧增。看看应用市场，其下载量在排行榜上占据靠前的位置，甚至有些占据榜首；各种分析和介绍的视频也是随处可见；NVIDIA 股价当场跳水……单从这些点，便可见其影响力之大，而且至少在接下来一段时间内还会继续上升。原谅我词穷，找不到更好的形容词了。

官方版量大管饱、便宜好用。网页版和 APP 目前是免费的，注册就能用（至少目前我还没找到对话时所谓剩余额度和充值入口）；API 是需要消耗额度的，但有免费赠送的额度，充值额度也很有性价比。

用了下，发现效果很不错。就拿春节期间我和 GPT4o-mini 随意写的“小说”举例：我随意地规划了一个故事情节，提了几点要求，加了点引导，让 GPT 写了个小说，然后把初稿给 Deepseek 评价和提建议；之后，我发自内心地放弃了我原来拟的大纲。写成那个样子的小说，它不仅把 GPT 写得漏洞百出的剧情~~（不过一次性写了近万字也属实难为它了）圆回来了，甚至还把主旨拔高了不少（有一种改作文发现其工作量比重写还大后选择后者的美）~~。

回到正题。由于其极高的访问量、不俗的性能，以及一些“特殊原因”，这个模型鼓舞了一些人，同样的也惹到了一些人，因此得到了各种意义上的“特殊关照”。如果你使用过它，应该会注意到，询问问题时它经常会提示“繁忙”。其他平台也陆续上线了这个模型，但我想总有些人~~（咽口水）~~不会优先考虑付费方案。

如果询问普通的内容，在网页版或 app 中使用 Deepseek 非常不错，但是一旦问一些敏感内容例如某些大伙想看的东西时，很容易被内容审查终止。这个模型还能接入一些代码编辑器、知识库、搜索等工具，但有些时候因数据性质所致，在线使用不太适合。

好在，Deepseek 提供了本地部署模型的方案，在本地部署则没有这些顾虑了（唯一的顾虑可能就是硬件配置）……

Ollama or LM Studio

比较项 LM Studio Ollama + WebUI
核心定位 面向非开发者的本地化桌面工具，提供一体化 GUI，闭源免费 面向开发者的命令行框架，可以直接使用，也可以搭配第三方 UI 使用，开源
使用难度 适合小白用户，支持 macOS、Windows 需命令行操作，图形化界面需另外部署安装，全平台支持
模型管理 连接 HuggingFace 模型库（国内访问不佳），直接下载 GGUF 格式模型 支持自定义 Modelfile，可微调模型，社区提供预训练模型

隐私与部署 完全离线运行，数据不联网，适合对隐私要求高的场景 本地部署，也可以内网穿透
适用场景 个人快速体验本地模型 开发者调试、企业私有化部署、多模型混合推理

Ollama 安装与配置

Ollama 是一个开源的 LLM（大型语言模型）工具，旨在提供可定制化的 AI 体验，支持用户自定义模型并进行本地部署。它的官网首屏中间只有一句话：Get up and running with large language models（启动并运行大型语言模型）。

首先进入 Ollama 官网 https://ollama.com/，点击Download按钮，选择适合自己设备平台的安装包下载。

速度实在有亿些些慢，如果你有 GitHub 加速工具，也可以去它的官方仓库 https://github.com/ollama/ollama 下载最新的 Release。

之后双击下载好的OllamaSetup.exe安装。有些 Windows 用户不喜欢将其安装在 C 盘，却发现安装程序没有让你选择安装路径的余地，可以这么操作：

下载模型

还是进入 Ollama 官网 https://ollama.com/，在顶部 Search models 框内输入你想要的模型的名称，如有则表明其支持。在这里，我们以 deepseek-r1 模型为例。

根据自己设备选择合适的模型：

  • 8G 内存：1.5B （1.1GB 左右大小）
  • 16G 内存：8B（4.9GB 左右大小）
  • 32G 内存：14B（9GB 左右大小）
  • 64G 内存：32B（20GB 左右大小）

其实这些都是蒸馏后的模型，数据量、“深度思考”能力、逻辑能力等都远不及 671B 那个原版的模型，但原版模型仅储存占用就 404GB，其对硬件要求也很高，一般家用电脑跑不起来。

图形界面

这样虽然可以直接使用，但对于大多数人而言，图形化界面更易于使用。好在，Ollama 有一系列的周边工具，提供了图形化的操作界面。目前网上大致有这些工具：

名称 核心亮点 适用场景
Open WebUI 企业级功能，支持知识库检索 团队协作、需结合内部文档的智能问答系统
LobeChat 多模态交互（图片/语音）、插件生态（联网搜索/代码执行），界面美观 个人全能助手、多模型切换（支持ChatGPT/Ollama混合使用）
Chatbox 轻量级跨平台，开箱即用，支持离线模式 教育演示、低资源运行模型
Enchanted 苹果生态原生优化，隐私优先 macOS、iOS 用户（追求无缝体验），有多设备同步的需求
NextJS UI 极简浏览器交互，Markdown渲染优化，资源占用低 开发者临时调试、追求极简设计的用户

下面这些工具是我已经使用过的：

Page Assist

Page Assist 是一个本地 AI 模型的 Web UI，项目地址为 https://github.com/n4ze3m/page-assist。作者提供了 Chrome 和 Firefox 扩展商店的链接（暂无 Edge）：

  • Chrome 扩展链接（需要“魔法”）
  • Firefox 扩展链接

受限于网络条件，此处以 Firefox 为例。安装后打开扩展，点击右上角的齿轮，进入设置，修改语言。之后回到主界面，在顶部选择模型，即可开始对话。

Chatbox

进入https://chatboxai.app/zh，下载适合自己设备的安装包，并安装软件。

在软件界面选择使用自己的 API Key 或本地模型，选择Ollama API，设置相应参数和选项，点击保存。后续想要修改设置，可以在软件左下角设置中修改。

AingDesk

项目链接：AingDesk。逛 GitHub 时发现的，使用前需要配置好 Ollama，若无则首次使用会提示“安装模型管理器”。

它的模型管理功能我比较喜欢（图形化，懒得敲命令）。在这里可以直接搜索 Ollama 支持的模型与相应的介绍链接，比较方便。（界面截图就不贴了）
