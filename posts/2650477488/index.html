博客二周年记

发表于2023-07-09|更新于2023-07-09|网站公告
|总字数:1.7k|阅读时长:5分钟|浏览量:

距离搭建网站已经过去约两年，是时候来回顾一下了。

跌宕起伏

大概是 2021 年起，因为自行了解了些 Web 相关知识，便学着网上一些人的教程去搭建网站。

当时我并不知道那些托管在 GitHub 上的静态站点是怎么做出来的（我竟天真地以为这是手动编辑 HTML 模板实现的），但考虑到多种因素（既没域名也不会用 Git）从而放弃，又考虑到费用问题，因此我的第一个站点是运行在那些“免费虚拟主机”上的。

早期的模板是从“站长素材”找来的，设计十分简朴，只有几个贴图（用于划分导航和正文区域），顶部有一个邮递马（驴？）车图案。因为有些久远，那个模板我找不到了，基于此编辑的所谓“博客”自然也找不到了（真就是手动创建链接的，所以搜索功能也自然是没精力实现的）。

长久来看太麻烦了，在一次机缘巧合之下我接触了一些开源 CMS 程序，其中修罗论坛算是我接触的继网盘后第二种网站程序。虽说是论坛系统，但我试着看了源码，发现其结构和实现方法没有想象中那么困难，一些基本的插件也都有，同时也可作为开发参考。

不过，它使用的技术有点老了，还有些漏洞（是靠社区第三方来修复的）。按理说，这些产品都是有官网的，再不济也有个官方渠道，况且看起来没那么冷门；可搜索到的第一个结果是非官方社区。那么官方社区呢？后来我才知道是作者心灰意冷关闭了官网……

失去了官方维护，其靠谱些我需要存疑了。那么还有什么呢？我尝试了解了 WordPress、Typecho 等其他基于 PHP 的系统，但都令我不大满意；Typecho 确实轻量，但维护频率有点过于低了，插件也没那么好找（现在好多了）。我也不是没想过 WordPress，但那种免费主机的配置运行它体验是好不到哪里去的。

也罢，还是拿修罗论坛练手，至少先“安顿”下来再说。

开发

接上节，最终我还是暂时选择修罗论坛来驱动我计划搭建的网站，XiunoBBS 也因此成为了我学习 Web 开发的第一个项目实例。

按照我自己的思路参考（Copy）了些其他大佬的网页特效与布局，最后定制化二开了一个主题，拿来自用。布局基本是模仿一个 Hexo 主题（那时我不知道这主题是 Butterfly），但受限于 Bootstrap4 框架与整体统一性，最后改得很奇怪，索性加一个 Column 再参考 Angulr、Musik 等模板（或者说是 Typecho 生态中与这模板仅外观相似的 Handsome 主题）再加以调整。

在不久前，OpenAI 推出了 ChatGPT。此前 AI 工具还没现在这么实用（或者说没什么知名又通用的 AI 工具），我碰到所有用到的 HTML、CSS、JavaScript 语法和效果都得去参考现有插件或主题的实现方法，或是翻阅 MDN 文档，又或是使用搜索引擎查找、在论坛里提问。从现在的角度来看这实在是太“原始”了。

在没有 GPT 的时候，我只能凭借搜索引擎去学习去转化；现在有了 GPT 之后，一些困难的问题都很容易得到解答了。虽然现在有些问题我还是会询问 GPT 的国内“镜像”工具，但我还是很感谢在它出现并火出圈之前的时光，在那段时间里，我培养了自己的搜索和转化的能力。

就像学习一样，如果是抱着求知的态度，那么学习者一般不会直接拿题目和参考答案学习，而是去教材中学习。基础是教材，而不是习题。我不大相信，拿着习题和答案推出来的知识点能比教材清晰。题目千变万化，而核心知识点却是不变的。拿着教材去研究解题，其中的思考历程远比拿着参考答案来得长，收获也更大。

同样的，有问题就问 GPT 固然是方便的，但久而久之会让自己失去追求解答的耐心，甚至会被误导——万一这个参考答案错了呢？还是那样，习题千变万化，而知识点却是不变的。如果你使用过这样的人工智能工具，会发现一些复杂的代码需求它很难一次处理好，终究还是要自己去调整的。

也许将来这样的 AI 工具会越来越方便，准确性也越来越高，没准那时使用它快速验证想法会变得十分普遍。那么灵感和微调的方法何来？前者靠“看”和积累，后者还是要自己去学习和转化。

重新开始

因为一些操作上的失误，导致先前那个站点崩了，只剩下少量仍以 Word 保存的草稿。同时我又没有及时备份的习惯，加上学习安排的问题，等我发现它无法访问时已经晚了。

恰好那时我正在尝试 SSG（Static Site Generation，静态站点生成）类的工具，并且我也认识到先前“魔改”修罗论坛时参考的模板的名称了，故花了一个晚上在某个知名的代码平台上成功托管了基于 Hexo 的站点。考虑到一些“历史遗留问题”，原站点将继续保留一段时间。

十分幸运的是，我不久前还成功申请到了一个免费的二级域名（虽然审核是真的慢），可以用 Cloudflare 管理，也可以绑定到那个托管的仓库上——域名访问的问题也解决了。

先试着运行一段时间，如果一切正常，之后便完全迁移过来了。

在丢失过往文章时我有些失落。好在那里的文章基本是测试性质的，少有的几篇还是“整活”为主。那时我不清楚我想要写什么，于是将以前老师布置的作文再改写一遍搬上来；也许是心境和见识不同了，写的时候有了新的意见。

将来会写些什么呢？也许是以我个人的学习经验为主，遇到问题、解决问题，并总结解决方法然后记录。既然个人博客有更高的自由度，也许将来会不仅限于经验记录部分……
