一键分析 Stable Diffusion 图像提示词：WD1.4 标签器安装与使用

发表于2024-10-02|更新于2024-10-02|经验记录
|总字数:2.5k|阅读时长:8分钟|浏览量:

中间有好一段时间没有用 SD 了，WD1.4 标签器更是很久未用，以致更新后都没注意到它没加载出来。查阅资料，进行修复，以此文记录。

Stable Diffusion 因为其开源的性质，在庞大的社区支持下，它已经成为一个强大的图像生成工具；随着图形化界面、民间的支持文档、不同模型的开发，其上手难度得到了进一步降低。

自打上手 AI 作图且基础配置完成后，“灵感”成了我最大的限制之一——准确来讲，是“如何用语言描述自己的灵感”。看着别人画出来的画如此好看，自己写的那提示词出的图马上就“不忍直视”了。而且对于 SD-WebUI，它的设置项很多，可控制的东西也很多，但只要稍微有点偏差（尤其是提示词一块），出图可能就截然不同了。“他到底用了什么 Tag？”

当然，用 SD 直接出的图一般是可以通过相应的工具解析出来，但如果这张图不是 SD 算法出的，又或是经过修改了呢？

WD14 是一个基于 SD-Webui 的标签插件（实际上应称之为“扩展”），也有 ComfyUI 版本。这个插件可以使用各种模型（如 DeepDanbooru）从已有的单个或多个图像文件中反推出来可能符合的 tag 标签（因此还是需要进行适当调整），还可以看到不同 tag 的权重。通过这些关键词可以便于我们了解图像的构成，并模仿着生成类似风格的图像，适合那些希望通过学习他人作品来提升自己创作水平的人。

本文面向使用 SD-WebUI 的用户。

安装

访问 https://github.com/toriato/stable-diffusion-webui-wd14-tagger 下载插件（虽然这个插件比较知名，但因为作者的身体原因，该仓库已经很久没有维护了，如果使用了 1.6.0 以上版本的 stable diffusion 以及相应的 SD-WebUI，那么直接使用该插件大概率是要报错的，后面会讲解决方法）。

嫌麻烦的可以直接试试它的第三方维护版：

  • picobyte/stable-diffusion-webui-wd14-tagger
  • akegarasu/sd-webui-wd14-tagger

建议直接安装这些维护分支的版本，它们已经解决了上述的因 SD-WebUI 主程序源码变动导致的报错，并进行了一定的优化。如果你选择安装上述的第三方维护分支的插件，那么后面修改文件一节可以直接跳过。

下文的安装方法选择其一即可（此处保留原版插件的地址，如有需要请自行调整）：

直接在 GitHub 点击绿色Code按钮，在Local选项卡中点击Download ZIP进行下载（自备加速器，例如 Watt Toolkit。顺带一提，目前新的测试版 Watt Toolkit 的加速脚本支持加速 HuggingFace 了，好耶、~）。

之后将插件解压到 SD-WebUI 中extensions目录下（带文件夹）（不会解压的请自行百度）。解压完后，该目录下应有一个名为stable-diffusion-webui-wd14-tagger的文件夹，里面是一些 Python 脚本和其他配置文件（夹）。

通过 SD-WebUI 安装

在前面那个下载地址中点击绿色Code按钮，点击Local选项卡中的Clone处HTTPS选项卡下方的类似于复制的按钮，复制 Git 链接。

之后启动 SD-WebUI，点击设置选项卡，点击从网址安装，将前面的 Git 链接填入插件的 git 仓库网址下方的文本框，点击安装后等待执行。若整个安装过程无误，则在完成后重启面板之后，顶部选项卡会出现WD1.4之类的字样。

使用

这里以一张 Pixabay 中的图片作为示例：

（截图是汉化后的 WebUI 中的截图，英文版可能需要自己对应，英文原版的选项卡的名称也不一定是 WD1.4 标签器）

在 WD1.4 标签器选项卡中选择想要分析的图像并上传，之后选择一个反推模型（一般使用 wd14-vit-v2.git），其他的保持默认即可。之后点击 Interrogate 按钮，插件会自动分析图像并生成一组提示词（速度因设备配置不同而异），完成后右侧会出现可能用到的 tag 及权重。

另外，用完后记得点击按钮卸载所有反推模型，避免出现一些意想不到的问题。

下面它的一些常用设置项：

项目 作用 使用场景
预设 使用已经配置好的参数组合，快速选择并应用它们来简化操作 常用的标签提取任务，节省时间
反推 从现有的标签中推断出可能的提示词 当你有一组标签但不确定如何生成相应的提示词时，可以使用反推功能
卸载反推所有模型 卸载所有加载的反推模型，以释放内存资源 在内存资源紧张时，或不再需要反推功能时
阈值 设置标签提取的敏感度，决定哪些标签会被提取出来 根据需要调整阈值，可以控制提取标签的数量和精度
附件标签 用于手动添加额外的标签，以补充自动提取的结果 当自动提取的标签不够全面或不准确时，可以手动添加补充
排除标签 指定不希望出现在结果中的标签 当某些标签不相关或不需要时，可以通过排除标签功能将其过滤掉
